# 🛡️ Prahari Web App - API Documentation

* **How it works:** Every protected request must include `connection_id` and `auth_code` in its payload.
* **Authentication injection:** If valid, the middleware dynamically attaches `user_id` to the request object, enabling controllers to identify the calling Prahari:
  ```php
  $user_id = $request->user_id; // Injected by verify.user middleware
  ```
* **Failure Response:** If credentials do not match or are missing:
  ```json
  {
      "status": false,
      "data": null,
      "message": "user not authenticated"
  }
  ```

## 📂 2. API Endpoint Directory

### 🔓 Public / Authentication Routes (`prefix: api/auth`)
| HTTP Method | Route Path | Controller Action | Purpose |
| :--- | :--- | :--- | :--- |
| **POST** | `/api/auth/register` | `AuthController@register` | Aadhaar-based simulated Prahari registration. |
| **POST** | `/api/auth/login` | `AuthController@login` | Standard Email/Password login (generates Sanctum token). |
| **POST** | `/api/auth/getconnectionId` | `AuthController@getConnectionId` | Requests a unique 32-character API Connection ID using pre-shared API Key. |
| **POST** | `/api/auth/requestOtp` | `AuthController@requestOtp` | Sends a 6-digit verification code to the registered mobile. |
| **POST** | `/api/auth/verifyOtp` | `AuthController@verifyOtp` | Validates OTP and returns the `auth_code` for subsequent requests. |

### 🔒 Protected Application Routes (`prefix: api/app` via `verify.user` middleware)
> ⚠️ **Note:** For every endpoint below, `connection_id` and `auth_code` must be passed in the request body (or request parameters).

| HTTP Method | Route Path | Controller Action | Purpose |
| :--- | :--- | :--- | :--- |
| **POST** | `/api/app/auth/logout` | `AuthController@logout` | Invalidates user's session and revokes connection `auth_code`. |
| **GET** | `/api/app/dashboard` | `DashboardController@index` | Fetch earnings, statistics, and recent cases/challans. |
| **GET** | `/api/app/profile` | `UserController@index` | View Prahari profile details, wallet balance, and status. |
| **PUT** | `/api/app/profile/update` | `UserController@updateProfile` | Update personal details (Name, Mobile, Bank Account). |
| **PUT** | `/api/app/profile/change-password` | `UserController@changePassword` | Change application password. |
| **GET** | `/api/app/cases` | `CaseController@index` | View list of cases reported by the Prahari (filter by status). |
| **POST** | `/api/app/cases` | `CaseController@store` | Report/Submit a new case with vehicle number, category, location, and media evidence. |
| **GET** | `/api/app/challans` | `ChallanController@index` | View generated challans, fine amounts, and statuses. |
| **GET** | `/api/app/wallet` | `WalletController@index` | Fetch current wallet balance and history of transactions. |
| **POST** | `/api/app/wallet/withdraw` | `WalletController@store` | Request a cash withdrawal from the wallet to a bank account. |

## 📄 3. Detailed Endpoint Reference

### 🔓 Public Authentication Endpoints

#### 1. Register Prahari
* **Route:** `POST /api/auth/register`
* **Description:** Creates a new `User` record along with a linked `Prahari` profile. The `prahari_id` is automatically generated by finding the latest ID starting with `PR` and incrementing it by 1 (defaulting to `PR1001` if no records exist). The Aadhaar number is validated but not stored.
* **Input Request:**
  | Field | Type | Validation Rules | Description |
  | :--- | :--- | :--- | :--- |
  | `name` | `string` | Required, Max 255 | Prahari's full name. |
  | `email` | `string` | Required, Email, Max 255, Unique in `users` | Account email address. |
  | `mobile` | `string` | Required, Max 15, Unique in `praharis` | Active mobile number. |
  | `aadhaar` | `string` | Required, Max 12 | Prahari's 12-digit Aadhaar Card number (used purely as authentication layer; not stored in database). |
  | `password` | `string` | Required, Min 8, Confirmed | Account password. |
  | `password_confirmation` | `string` | Required | Password confirmation match. |

* **Success Response (`200 OK`):**
  ```json
  {
      "success": true,
      "message": "Registration successful",
      "data": {
          "user": {
              "id": 15,
              "name": "Jane Doe",
              "email": "jane.doe@example.com",
              "updated_at": "2026-05-27T10:15:30.000000Z",
              "created_at": "2026-05-27T10:15:30.000000Z"
          }
      }
  }
  ```
* **Validation Error Response (`422 Unprocessable Content`):**
  ```json
  {
      "success": false,
      "errors": {
          "email": [
              "The email has already been taken."
          ],
          "mobile": [
              "The mobile has already been taken."
          ]
      }
  }
  ```

#### 3. Establish Connection ID
* **Route:** `POST /api/auth/getconnectionId`
* **Description:** Requests a unique 32-character API Connection ID using the server's API key.
* **Input Request:**
  | Field | Type | Validation Rules | Description |
  | :--- | :--- | :--- | :--- |
  | `app_api_key` | `string` | Required | Shared backend API key matching `config('app.api_key')`. |

* **Success Response (`200 OK`):**
  ```json
  {
      "status": 200,
      "message": "connection id generated successfully",
      "data": {
          "connection_id": "8a7fbc1d9e2a3c4f5b6d7e8f9a0b1c2d"
      }
  }
  ```
* **Failure Response (`401 Unauthorized`):**
  ```json
  {
      "status": 401,
      "message": "Invalid api key"
  }
  ```

#### 4. Request OTP Verification Code
* **Route:** `POST /api/auth/requestOtp`
* **Description:** Initiates standard 6-digit OTP generation and dispatches it via SMS Service to the registered user.
* **Input Request:**
  | Field | Type | Validation Rules | Description |
  | :--- | :--- | :--- | :--- |
  | `connection_id` | `string` | Required, must match database | active connection session ID. |
  | `mobile` | `string` | Required, string, min 10, max 15, exists in `praharis` table | The registered Prahari mobile number. |

* **Success Response (`200 OK`):**
  ```json
  {
      "status": 200,
      "message": "OTP sent successfully",
      "data": {
          "otp_sent": true
      }
  }
  ```
* **Failure Response (`400 Bad Request`):**
  ```json
  {
      "status": 400,
      "message": "Invalid Connection. Please generate Connection Id",
      "data": null
  }
  ```

---

#### 5. Verify OTP Code
* **Route:** `POST /api/auth/verifyOtp`
* **Description:** Confirms the 6-digit OTP and generates an `auth_code` for API consumption.
* **Input Request:**
  | Field | Type | Validation Rules | Description |
  | :--- | :--- | :--- | :--- |
  | `connection_id` | `string` | Required | Session connection ID. |
  | `mobile` | `string` | Required | Prahari mobile number. |
  | `otp` | `string` | Required | 6-digit SMS verification code. |

* **Success Response (`200 OK`):**
  ```json
  {
      "status": true,
      "message": "Otp Verified Successfully",
      "data": {
          "connection_id": "8a7fbc1d9e2a3c4f5b6d7e8f9a0b1c2d",
          "auth_code": "kF9qL2vX4m"
      }
  }
  ```
* **Invalid OTP Response (`200 OK` with status `false`):**
  ```json
  {
      "status": false,
      "message": "Invalid OTP...",
      "data": null
  }
  ```
* **OTP Expired Response (`200 OK` with status `false` - Expiry is 5 Minutes):**
  ```json
  {
      "status": false,
      "message": "OTP Expired...",
      "data": null
  }
  ```

#### 12. Submit New Case
* **Route:** `POST /api/app/cases`
* **Description:** Creates a new traffic/community violation report with local metadata and optional media evidence.
* **Input Request:**
  > 📝 **Content-Type Note:** Because this endpoint accepts files, the request **MUST** be sent as `multipart/form-data`.

  | Field | Type | Validation Rules | Description |
  | :--- | :--- | :--- | :--- |
  | `connection_id` | `string` | Required | Connection ID. |
  | `auth_code` | `string` | Required | Verification Code. |
  | `case_category_id`| `integer`| Required, exists in `case_categories.id` | Category code of violation. |
  | `vehicle_number` | `string` | Required, Max 20 | License plate of the target vehicle. |
  | `location` | `string` | Required | GPS Coordinates or text address of the scene. |
  | `evidence` | `file` | Nullable, File, mimes: `jpg,jpeg,png,mp4,mov`, Max 10MB | Image or Video file capturing the incident. |

* **Internal ID Generation Details:** 
  The case ID is auto-incremented dynamically based on the last record's code (e.g. `CASE2309`, `CASE2310`, `CASE2311`, etc.).
  The uploaded evidence is moved directly to the public directory: `public/assets/video/` and saved as `{CASE_ID}.{extension}`.

* **Success Response (`200 OK`):**
  ```json
  {
      "success": true,
      "message": "Case created successfully",
      "data": {
          "case_id": "CASE2313",
          "prahari_id": 1,
          "case_category_id": 2,
          "vehicle_number": "MH12AB9999",
          "location": "Sunset Boulevard Crossing",
          "evidence": "assets/video/CASE2313.mp4",
          "status": "Open",
          "updated_at": "2026-05-27T10:30:00.000000Z",
          "created_at": "2026-05-27T10:30:00.000000Z",
          "id": 5
      }
  }
  ```

#### 15. Request Cash Withdrawal
* **Route:** `POST /api/app/wallet/withdraw`
* **Description:** Submit a request to withdraw funds to a bank account. Note that this immediately decrements the wallet balance in the database and creates a transaction entry with status `pending`.
* **Input Request:**
  | Field | Type | Validation Rules | Description |
  | :--- | :--- | :--- | :--- |
  | `connection_id` | `string` | Required | Connection ID. |
  | `auth_code` | `string` | Required | Verification Code. |
  | `amount` | `numeric` | Required, Min 1, Max `wallet_balance` | Amount of money to withdraw. |
  | `bank_account` | `string` | Required | Target bank account number. |

* **Transaction ID Generation Note:**
  Withdrawal IDs are automatically generated in sequence on model creation (e.g. `W0001`, `W0002`, `W0003`, etc.).

* **Success Response (`200 OK`):**
  ```json
  {
      "success": true,
      "message": "Withdrawal request submitted successfully",
      "data": {
          "prahari_id": 1,
          "amount": "500",
          "bank_account": "123456789012",
          "status": "pending",
          "withdrawal_id": "W0002",
          "updated_at": "2026-05-27T10:45:00.000000Z",
          "created_at": "2026-05-27T10:45:00.000000Z",
          "id": 3
      }
  }
  ```
* **Validation Error Response (`422 Unprocessable Content`):**
  ```json
  {
      "success": false,
      "errors": {
          "amount": [
              "The amount must not be greater than 3500.00."
          ]
      }
  }
  ```
